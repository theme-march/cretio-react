React Installation

Please follow the instructions to see how you can install the project on your local machine:

  1. For local host -
    1. Open your terminal / command prompt in the project root
    2. Run: npm install
    3. Run: npm run dev (to start the dev server at http://localhost:5173)

    1. Open your terminal
    2. Run: npm run build (this will generate the dist folder in the root directory)

Basic Site Settings

Change Site Title and Favicon

To change your Site title Favicon and Page Titles; open the cretio-react-documentation in your editor and go to the location by following screenshot which are given bellow:

  1. Update Site Title

    Navigate to the root folder and open index.htmlOpen the file . Locate the <title> tag inside the <head> section and update the text to your desired site name.

  2. Manage Favicon

    Replace the default icon file at srcassetsimglogofavicon.svgReplace the file with your custom icon.

Change Logo

To update the logos across the entire site; Replace the existing files at srcassetsimglogo . If you want to rename the files or extensions, update their paths in srcdataJsonsiteSettings.json under the "logos" key.

Change Page Title

To change the main heading (title) of an individual page, navigate to the specific page file and update the title prop in the Breadcrumb or Hero component.

  1. Locate the Page Component:

    Go to srcpages and open the file for the page you want to edit (e.g., About.tsx, Services.tsx).

  2. Update the Title Prop:

    Locate the Breadcrumb component and change the title and highlightWords (optional) to your desired text.

    <Breadcrumb
    title="Your New Page Heading"
    highlightWords={["New", "Heading"]}
    ...
/>

Customize Menu

The navigation menu is dynamically generated from a JSON configuration file. To manage your menu items, follow these steps:

  1. Locate the Configuration File

    Navigate to: srcdataJsonnavItemsList.jsonOpen the file

  2. Understand the Structure

    Each menu item is an object with the following properties:

    • title: The text displayed in the menu.
    • href: The URL path for the link.
    • subMenu (Optional): An array of objects for dropdown items.
  3. Adding a Submenu (Dropdown)

    To create a dropdown, add a "subMenu" array to any menu item. For example:

    {
  "title": "Services",
  "href": "/services",
  "subMenu": [
    { "title": "Service 1", "href": "/service-1" },
    { "title": "Service 2", "href": "/service-2" }
  ]
}

Customize Global Settings & Footer

The site's branding, contact information, and footer content are managed via a single global configuration file.

  1. Locate the Configuration File

    Navigate to: srcdataJsonsiteSettings.jsonOpen the file

  2. Customizable Sections
    • logos: Update paths for the dark, white, and footer-specific logos.
    • cta: Customize the "Work Together" section title, button text, and contact email.
    • company: Update the company description text.
    • contact: Manage the phone number, address, and click-to-call links.
    • socials: Update the social media icons (using Flaticon classes) and their respective profile links.
    • gallery: Update the list of image paths for the small footer gallery.

Customize Blog data

To manage your blog posts, you need to update the central JSON data file. Follow these steps:

  1. Locate the Data File

    Navigate to: srcdataJsonblogsData.jsonOpen the file

  2. Understand the Blog Structure

    Each blog post is represented as an object with the following properties:

    • id: A unique numeric identifier for the post.
    • image: The path to the blog thumbnail image (e.g., src/assets/img/blogs/blog-1.png).
    • author: The name of the post author.
    • date: The publication date in DD/MM/YYYY format.
    • title: The main headline of the blog post.
    • description: A short summary or excerpt of the post content.
  3. Adding New Posts

    Simply append a new object to the array in the JSON file. Ensure you provide a unique id for each new entry.

Customize Services list data

Services data is managed through a structured JSON file that supports multiple layout variants. To customize your services, follow these steps:

  1. Locate the Data File

    Navigate to: srcdataJsonservicesData.jsonOpen the file

  2. Customize Section Info

    Use the sectionInfo object to update the global header for the services section (Title, Description, Short Caption, and Button Text).

  3. Manage Main Services

    Each service in the mainServices array includes:

    • id: A unique numeric index.
    • title: The name of the service.
    • description: A brief overview of the service.
    • image: The background/hover image path.
    • features: An array of strings for key benefits or features.
  4. Specialized Lists

    Depending on your page variant, you may also need to update seoServices, brandingServices, or digitalMarketingServices arrays.

Customize Team lists data

The team section is data-driven and supports multiple layouts for different project variants. To customize your team members, follow these steps:

  1. Locate the Data File

    All team information is stored in: srcdataJsonteamMembersData.jsonOpen the file

  2. Understand the Team Variants

    The JSON file contains two primary lists: standardTeam (Creative Agency) and marketingTeam (Marketing Agency).

  3. Member Entry Fields

    Each member profile uses the following structure:

    {
  "id": 1,
  "name": "Alex Johnson",
  "role": "Creative Director",
  "image": "src/assets/img/team/team-1.png"
}
  4. Image Asset Placement

    Upload new team member photos to src/assets/img/team/ and update the image path in the JSON file accordingly.

Customize Portfolio data

The portfolio section uses Isotope for dynamic filtering. To manage your projects and filter categories, follow these steps:

  1. Locate the Data File

    Navigate to: srcdataJsonprojectsData.jsonOpen the file

  2. Configuring Filter Categories

    In the categories array, define your filter buttons. The filter property must be a CSS selector (e.g., .isf-uiux).

  3. Manage Project Entries

    Each project in the mainProjects array includes:

    • id: A unique identifier.
    • title: The project name.
    • category: The display category name.
    • image: The project thumbnail path.
    • tags: CSS classes that must match the filter selector (without the dot).
    • colClass: The Bootstrap column width (e.g., col-md-6).
  4. Layout Variations

    Update gridProjects or sliderProjects if you are using those specific layout variants on different pages.

    // Category Filter
{ "name": "UI/UX", "filter": ".isf-uiux" }

// Project Entry
{ 
  "id": 1, 
  "title": "EcoHome Campaign", 
  "category": "Web App", 
  "tags": "isf-uiux isf-webapp",
  "image": "path/to/img.png" 
}

Error Pages

The 404 Error page is a standalone component. To customize its content, follow these steps:

  1. Direct Component Access

    Navigate to: srcpagesNotFound.tsxOpen the file

  2. Text Customization

    Update the following JSX elements to change the page messages:

    • <h2 className="error-message">: Update the main "Page Not Found!" headline.
    • <p className="error-description">: Customize the detailed error description text.
    • <Link>: Update the "Back To Home" button text and the to="/" path if needed.
  3. Visual Customization

    The "4!4" display uses a combination of text and an SVG exclamation mark. You can modify the SVG code or the surrounding className="number" spans to change the look and feel.

Customize Header

The header is a dynamic component that manages navigation, branding, and global settings. To customize it, follow these steps:

  1. Core Component

    The layout and logic are defined in srclayoutHeader.tsxOpen the file.

  2. Navigation Menu

    Manage all menu items and submenus by editing: srcdataJsonnavItemsList.jsonOpen the file. The header automatically renders this list.

  3. Logos & Branding

    The header pulls the Dark and White logos from the logos section in srcdataJsonsiteSettings.jsonOpen the file.

  4. CTA Button

    To change the "Start Project" button text or its destination link, modify the <Link> component within Header.tsx (look for the custom-button class).

  5. Offcanvas Menu

    The slide-out menu (triggered by the dots icon) displays your description, gallery, contact info, and socials by pulling data directly from srcdataJsonsiteSettings.jsonOpen the file.

SASS Customization

The project's visual style is architected using SASS for modularity and ease of maintenance. To customize the styling, follow these steps:

  1. The Style Entry

    The primary bridge between React and SASS is srcindex.scssOpen the file. This file imports all global plugins and the main theme stylesheet.

  2. Global Variables (Colors & Typography)

    For quick branding updates, navigate to srcassetssassdefaultOpen the folder:

    • _color_variable.scss: Edit this file to update the theme's primary, secondary, and background colors.
    • _typography.scss: Use this file to manage font families, weights, and global font sizes.
  3. SASS Folder Structure

    The srcassetssassOpen the folder directory is organized into functional modules:

    • common/: Shared utilities and layout basics.
    • components/: Styles for specific UI elements like modals, buttons, and sections.
    • shortcode/: Advanced template modules and UI blocks.
  4. Compiling Styles

    The theme uses a pre-compiled style.css for performance, but you should always modify the .scss files. The build process will automatically handle the compilation during development.

App Development Guide

Basic workflow for extending the template and changing global styles:

  1. Adding New Pages

    To add a new page:
    1. Create a .tsx file in src/pages/.
    2. Import it in src/App.tsx.
    3. Add a new <Route path="/your-path" element={<YourPage />} /> inside the <Routes> component.

  2. Updating Google Fonts

    Navigate to src/assets/sass/default/_typography.scss and replace the @import URL with your chosen Google Font. Then, update the font-family variables in the same file.
